# Scrum Board Standards

## Purpose
Scrum boards provide visibility into
sprint execution and work-in-progress.

## Core Principles
- One board per team
- Clear workflow stages
- Visible blockers and impediments

## Good Practices
- Keep columns aligned with workflow
- Limit work in progress
- Review board health regularly

## Common Mistakes
- Overloading boards with custom fields
- Using boards as reporting tools only
- Allowing stale or inactive tickets
